The Problem: Slow Spring Test Suites

⏱

Expensive Context Startup

Each unique context configuration triggers a full Spring context creation — 2 to 10 seconds each

👁

Hidden Duplicates

Tiny config drifts between tests silently create separate contexts that could be shared

💣

@DirtiesContext Overuse

Destroys the cached context after each test, forcing expensive recreation for every method

😶

No Visibility

Spring's context cache is a black box — no built-in tools to see what's being reused or wasted

Setup in 3 Steps

1

Add the test dependency

Works with Maven and Gradle

2

Register via spring.factories

Add the listener + initializer to
src/test/resources/META-INF/spring.factories

3

Run your tests

mvn test or gradle test — report appears at
target/spring-test-profiler/latest.html

<!-- pom.xml --> <dependency> <groupId>digital.pragmatech.testing</groupId> <artifactId>spring-test-profiler</artifactId> <version>0.0.17</version> <scope>test</scope> </dependency>
# src/test/resources/META-INF/spring.factories org.springframework.test.context.TestExecutionListener=\ digital.pragmatech.testing.SpringTestProfilerListener org.springframework.context.ApplicationContextInitializer=\ digital.pragmatech.testing.diagnostic\ .ContextDiagnosticApplicationInitializer
